# DEVELOPER NOTES

## Схема проекта

```
Service-Repository Pattern
┌────────┐   ┌────────┐   ┌────────────────┐   ┌────────┐   ┌─────────────┐   ┌────────┐    ┌───────────┐
|        | → |        | → |                | → |        | → |             | → |        | → (             )
| Клиент |   | Router |   | Pydantic схема |   | Сервис |   | Репозиторий |   | Модель |   ( База данных )
|        | ← |        | ← |                | ← |        | ← |             | ← |        | ← (             )
└────────┘   └────────┘   └────────────────┘   └────────┘   └─────────────┘   └────────┘    └───────────┘
```

## Назначение файлов

Router - механизм для направления запроса к нужному обработчику
Pydantic схема - валидация входящих данных
Сервис - реализует бизнес-логику
Репозиторий - чистый доступ к базе данных
Модель - описание структуры таблицы базы данных
Pydantic ответ - унифицированный формат ответа клиенту

## Последовательность разработки файлов и команды для их создания

1. Модель
2. Схемы Pydantic
3. Репозиторий
4. Сервис
5. Роутер

### Запуск приложения

uvicorn app.main:app --reload

pytest -v --cov=app --cov-report=term-missing --cov-report=html
